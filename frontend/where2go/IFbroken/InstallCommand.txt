Complaints
- If your branch stops working, consider trying someone else's branch as base setup
- build breaking look at versions, mainly java
- Using a more advanced version of java (jdk 25) can run you into a bunch of trouble. i got jdk 21 or 17 i dont remember anymore
- If you reach the point where the error asks you to change a gradle file or extremely specific lines of files, cancel/revert/go back you are lost. change java version instead
- Deleting and recloning the entire repo is perfectly valid. Just annoying that some folders require you to delete the subfolders first because the admin perm blocks you. Even though you are the admin
- adb.exe breaking might be linked to the emulator. Go read More troubleshooting/In android emulator
